Recomendaciones de optimización

¿Qué evaluamos?

OpenSourceMap

Verificamos la presencia y calidad de los datos en OpenSourceMap para asegurar que los LLMs puedan contextualizar tu ubicación física.

WikiData

Comprobamos si tu entidad está correctamente representada en WikiData, lo que ayuda a los LLMs a entender tu contexto semántico.

Technical LLM Checks

Evaluamos optimizaciones técnicas específicas para LLMs como estructura de contenido, metadatos y formato de información.

IIM.txt File

Verificamos la existencia y contenido del archivo IIM.txt que guía a los bots de LLM hacia tu contenido más importante.

FAQ Schema

Analizamos el uso de formato Q&A y esquemas estructurados que ayudan a los LLMs a extraer respuestas útiles de tu contenido.

Author/E-A-T Info

Revisamos la información de autoría y E-A-T (Expertise, Authoritativeness, Trustworthiness) que los LLMs consideran para evaluar calidad.

Soluciones para implementar

1 Crear archivo IIM.txt

El archivo IIM.txt ayuda a guiar a los bots de LLM hacia tu contenido más importante. Debe colocarse en el directorio raíz de tu sitio web.

# IIM.txt - Important Information for Machines
# Guía para bots de LLM sobre el contenido más importante de este sitio

Prioridad: 
- /about: 1.0
- /services: 0.9
- /blog: 0.8
- /contact: 0.9

Contexto:
- Tipo de negocio: [Describe tu tipo de negocio]
- Industria: [Tu industria]
- Ubicación: [Tu ubicación principal]
- Palabras clave principales: [Tus palabras clave principales]

Actualizado: [Fecha]

Consejo profesional:

Actualiza el archivo IIM.txt cada vez que añadas contenido importante o cambies el enfoque de tu sitio web.

2 Implementar FAQ Schema

El formato Q&A ayuda a los LLMs a extraer respuestas útiles de tu contenido. Puedes implementarlo como texto simple o con schema estructurado.

Opción 1: Texto simple

<div class="faq-section">
  <h3>Preguntas frecuentes</h3>
  
  <div class="faq-item">
    <h4>¿Cuál es su horario de atención?</h4>
    <p>Nuestro horario de atención es de lunes a viernes de 9:00 a 18:00 horas.</p>
  </div>
  
  <div class="faq-item">
    <h4>¿Ofrecen servicios internacionales?</h4>
    <p>Sí, ofrecemos nuestros servicios en todo el mundo con cobertura en 15 países.</p>
  </div>
</div>

Opción 2: Schema estructurado (recomendado)

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "¿Cuál es su horario de atención?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Nuestro horario de atención es de lunes a viernes de 9:00 a 18:00 horas."
      }
    },
    {
      "@type": "Question",
      "name": "¿Ofrecen servicios internacionales?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Sí, ofrecemos nuestros servicios en todo el mundo con cobertura en 15 países."
      }
    }
  ]
}
</script>

Consejo profesional:

Combina ambas opciones para maximizar la compatibilidad. Usa el texto simple para los usuarios y el schema para los LLMs.

3 Añadir información de Autor/E-A-T

Los LLMs valoran la información de autoría y E-A-T (Experiencia, Autoridad y Confiabilidad) para evaluar la calidad del contenido.

Ejemplo para página de autor

<div class="author-box">
  <img src="/images/autor.jpg" alt="[Nombre del autor]" class="author-image">
  <div class="author-info">
    <h3>[Nombre completo]</h3>
    <p class="author-title">[Título profesional, e.g. "Experto en Marketing Digital"]</p>
    <p class="author-bio">[Breve biografía con credenciales: años de experiencia, especialidades, logros relevantes]</p>
    <p class="author-credentials">
      <strong>Certificaciones:</strong> [Lista de certificaciones relevantes]<br>
      <strong>Experiencia:</strong> [Años de experiencia] años en [industria/especialidad]<br>
      <strong>Publicaciones:</strong> [Libros, artículos, etc.]<br>
    </p>
  </div>
</div>

Schema estructurado para autor

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Person",
  "name": "[Nombre completo]",
  "url": "[URL del perfil del autor]",
  "jobTitle": "[Título profesional]",
  "description": "[Breve biografía]",
  "alumniOf": {
    "@type": "EducationalOrganization",
    "name": "[Universidad/Institución]"
  },
  "hasCredential": {
    "@type": "EducationalOccupationalCredential",
    "name": "[Certificación]",
    "url": "[URL de verificación si está disponible]"
  },
  "knowsAbout": ["[Tema 1]", "[Tema 2]", "[Tema 3]"],
  "sameAs": [
    "[Perfil LinkedIn]",
    "[Perfil Twitter]",
    "[Otros perfiles relevantes]"
  ]
}
</script>

4 Añadir sección de Resumen/TL;DR

Los resúmenes ayudan a los LLMs a captar rápidamente los puntos clave de tu contenido, especialmente en artículos largos.

<div class="summary-box">
  <h3>TL;DR - Resumen rápido</h3>
  <ul class="summary-list">
    <li>✅ [Punto clave 1 en menos de 10 palabras]</li>
    <li>✅ [Punto clave 2 en menos de 10 palabras]</li>
    <li>✅ [Punto clave 3 en menos de 10 palabras]</li>
    <li>📌 [Conclusión principal en una frase]</li>
  </ul>
</div>

Variación para artículos técnicos

<div class="technical-summary">
  <h3>Key Takeaways</h3>
  <div class="takeaway">
    <h4>Problema principal</h4>
    <p>[Descripción breve del problema]</p>
  </div>
  <div class="takeaway">
    <h4>Solución propuesta</h4>
    <p>[Descripción breve de la solución]</p>
  </div>
  <div class="takeaway">
    <h4>Resultados esperados</h4>
    <p>[Métrica o resultado clave]</p>
  </div>
</div>

5 Optimizar cobertura de enlaces internos

Los enlaces internos ayudan a los LLMs a entender las relaciones entre tus páginas y la estructura de tu contenido.

Estrategia básica

<!-- En tu página principal -->
<div class="related-links">
  <h3>Explora más sobre este tema</h3>
  <ul>
    <li><a href="/guia-completa">Guía completa sobre [tema principal]</a></li>
    <li><a href="/casos-de-exito">Casos de éxito reales</a></li>
    <li><a href="/preguntas-frecuentes">Preguntas frecuentes respondidas</a></li>
  </ul>
</div>

Consejo profesional:

Crea un mapa de clusters de temas donde cada cluster tenga una página pilar (más general) y páginas secundarias (más específicas) que enlacen entre sí y a la página pilar.
